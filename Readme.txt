工具类大多采用了static方法的形式，会常驻内存，所以使用时请有选择的使用，不要整个library导入。
    static方法和单例模式的使用原则：
    对于没有属性/状态的类，可以用这两种方式。对于频繁使用、一定使用的类用static，对于不太频繁、不一定会用的类用单例。
    static方法和单例模式的对比：
    （1）内存
    static方法会在类加载时候初始化，并常驻内存，但是只分配这一次内存；单例会在使用时才初始化，也是只占用一个对象的内存，但是考虑到单例的出现原因也是
    因为对于会多次调用的类只保存一个对象，这个单例实际会长期占用内存。相比之下，单例只是稍占优势。
    （2）速度
    因为static方法常驻内存，在使用上便有了速度优势，而单例怎么也得需要实例化一次。
